📋 FLUJO ESTÁNDAR DE TESTING:

  PASO 1: Antes de Commit (5 min)
  ─────────────────────────────────
  $ cd gf-studio
  $ node tests/test-smoke.js

  Si ves "SMOKE TEST PASSED" → puedes hacer commit
  Si ves errores críticos → corregir antes de commit

  ─────────────────────────────────

  PASO 2: Durante Desarrollo (20 min)
  ─────────────────────────────────
  1. Abre tests/test-mobile.html en navegador
  2. Selecciona dispositivo (iPhone 14, Pixel 7, iPad)
  3. Prueba la feature nueva
  4. Si ves borde rojo = overflow → corregir

  ─────────────────────────────────

  PASO 3: Pre-Release (60 min)
  ─────────────────────────────────
  1. node tests/test-smoke.js
  2. Abre tests/test-checklist.html
  3. Marca tests: Pass (✓), Fail (✗), Skip (⊘)
  4. Objetivo: 70%+ PASS
  5. Exporta resultados (botón "Copiar al portapapeles")
  6. Verifica Lighthouse > 85 en Vercel preview
  7. Prueba en navegadores reales

  ─────────────────────────────────

  PASO 4: Deploy
  ─────────────────────────────────
  Si todos los pasos pasaron:
  $ git add -A
  $ git commit -m "chore: release v1.x.x"
  $ git push origin main
  → Vercel auto-deploya

❓ FAQ RÁPIDAS:

  P: ¿Por dónde empiezo?
  R: Abre tests/index.html en navegador → sigue instrucciones

  P: ¿Cuánto tiempo lleva hacer testing completo?
  R: ~90 minutos (5 smoke + 20 mobile + 60 checklist + 5 lighthouse)

  P: ¿Necesito instalar algo?
  R: No. Node.js debe estar instalado para test-smoke.js (npm test)

  P: ¿Se guarda mi progreso del checklist?
  R: Sí, en localStorage del navegador (automáticamente)

  P: ¿Puedo resetear el checklist?
  R: Sí, botón "Resetear" en tests/test-checklist.html

  P: ¿Cómo exporto los resultados?
  R: tests/test-checklist.html → botón "Copiar al portapapeles"

  P: ¿Qué pasa si fallo algún test?
  R: Ve a TEST-PLAN.md → sección correspondiente → sigue pasos
